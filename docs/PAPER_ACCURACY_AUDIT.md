## Executive Verdict

**Mostly accurate, ready for internal supervisor review and Zenodo / preprint posting after a small number of additional targeted fixes (mostly to Section VI-A — macOS scanner — and Section IV-A — event-type count).** The major Ed25519 / ECDSA P-256 conflation, the field-name inventions in Section VIII-D, the 7- vs 8-category mismatch in Figure 1, the Swift test-file reference, and the field-count slip in Section IV-C have all been correctly fixed in the current draft. Validation numbers (327 Node, 33 Rust, 11 .NET, 16 Linux smoke, 30 Linux audit assertions, 371 total) re-verify cleanly today.

Remaining issues are smaller but should be fixed before posting:

1. **CONTRADICTION (new finding).** Section VI-A states the macOS scanner "queries `CGWindowListCopyWindowInfo` with the `kCGWindowListOptionAll` mask". The actual code (`AffinityScanner.swift:32`) uses `[.optionOnScreenOnly, .excludeDesktopElements]` instead.
2. **UNSUPPORTED (new finding).** Section VI-A states "Starting with macOS 15, ScreenCaptureKit provides richer metadata; the scanner adapts its query strategy based on OS version". The macOS daemon source contains **no** `ScreenCaptureKit` import, no `SCShareableContent` usage, and no `#available(macOS …)` branch. The scanner is single-path and uses CoreGraphics only. The prior audit flagged this in "Optional Improvements" and recommended softening; it has not been addressed.
3. **NEEDS NARROWING (carried over).** Section IV-A still says "18 defined academic event types". `src/academic/academicEvents.js` actually defines **41** event constants. Approximately 18 of those are pure browser-side behavioural events, but the paper text as written is misleading.
4. **MINOR.** Section X-B says "research prototype at version 0.4.18". The latest git tag and `CHANGELOG.md` head are indeed `v0.4.18-stage-2-8-linux-closeout`, but `package.json` still reads `0.1.0`. Either bump `package.json` or footnote that the version is the tagged research-release version, not the package version.
5. **MINOR.** Section IV-B item (1) says the Windows / Linux on-disk identity is "an OS-native identity file at mode `0600` in a `0700` directory". I did not re-grep the Windows/Linux identity-file permission code in this pass; verify against `tools/simurgh-daemon-windows/src/SimurghDaemon.Windows/WindowsIdentityStore.cs` and `tools/simurgh-daemon-linux/src/identity.rs` before submission.

With these corrections applied, the paper is suitable for: internal supervisor review → Zenodo preprint (companion DOI to the Invisible Window paper) → EdArXiv-OSF + arXiv (cs.CR) → academic-integrity / usable-security workshop submission.
